# 🏰 小主人童话 — MVP v0.1 规范

## Why
为 3-12 岁孩子的父母提供一个 AI 驱动的个性化童话故事生成平台。每个孩子都是独特的，他们的睡前故事也应该是独一无二的。当前市场上缺乏结合孩子真实特征、成长需求与 AI 生成能力的定制化童话内容平台。

## What Changes
- 新建家长端 Web 应用（React + TypeScript）
- 孩子档案管理系统，支持 AI 结构化标签提取
- AI 故事生成引擎集成（故事文本 + 插画）
- 故事管理（生成、预览、推送、收藏）
- 社区故事市场（浏览、搜索、分享）
- Token 经济基础模型
- 简易儿童端 Web 查看页面
- 支持 Render 部署与前后端环境变量化配置
- 支持本地 SQLite / 生产 PostgreSQL 的 Prisma 多环境切换

## Impact
- 受影响规范：无（全新项目）
- 受影响代码：全项目新建

---

## ADDED Requirements

### Requirement: 项目初始化与技术选型
系统 SHALL 基于 React + TypeScript + Vite 构建前端应用，使用 TailwindCSS 作为样式方案，后端使用 Node.js + Express/Next.js API Routes。

#### Scenario: 项目成功初始化
- **WHEN** 开发者执行项目初始化命令
- **THEN** 项目目录结构完整，包含前端、后端、数据库模型、公共配置等基础模块

---

### Requirement: 孩子档案管理
系统 SHALL 允许家长创建和管理多个孩子的档案，包括年龄、性别、性格描述、当前需求等自然语言描述，并由 AI 自动提取结构化标签。

#### Scenario: 创建孩子档案
- **WHEN** 家长输入孩子的自然语言描述（如"5岁男孩，活泼好动，最近害怕黑暗"）
- **THEN** 系统保存档案并调用 AI 提取结构化标签（年龄:5, 性别:男, 性格:活泼, 关注点:怕黑）

#### Scenario: 管理多个孩子
- **WHEN** 家庭有多个孩子
- **THEN** 家长可以创建、编辑、切换、删除不同孩子的档案

#### Scenario: 孩子画像持续学习
- **WHEN** 家长多次为同一孩子生成故事
- **THEN** 系统基于历史故事主题更新孩子画像标签权重

---

### Requirement: AI 故事生成
系统 SHALL 支持家长通过 AI 采访式引导逐步表达故事需求，系统结合孩子画像、故事风格、画家风格和教育目标生成个性化故事文本与配套插画。

#### Scenario: AI 采访式引导生成故事
- **WHEN** 家长进入创建故事页
- **THEN** 系统默认展示 AI 对话式采访入口，而不是传统整页表单
- **AND** AI 逐步引导家长确认孩子、描述事件情境、成长目标和风格偏好
- **AND** 在摘要确认后触发故事生成

#### Scenario: 采访结果映射为生成请求
- **WHEN** 家长完成采访对话
- **THEN** 系统将对话中收集的孩子、事件、教育目标、故事氛围和插画偏好整理为故事生成请求
- **AND** 继续复用现有故事生成链路与预览能力

#### Scenario: 故事风格选择
- **WHEN** 家长通过 AI 采访流生成故事
- **THEN** 系统在对话中收集故事风格偏好和插画风格偏好（如睡前、冒险、水彩、卡通等）

#### Scenario: 预览与确认
- **WHEN** 故事生成完成后
- **THEN** 家长可预览故事全文 + 插画，确认后推送到孩子端，或重新生成

#### Scenario: 无孩子档案时拦截采访流
- **WHEN** 家长进入创建故事页但尚未添加孩子档案
- **THEN** 系统优先提示先创建孩子档案
- **AND** 不进入正式采访消息流

#### Scenario: 按年龄自适应故事篇幅
- **WHEN** 系统根据孩子年龄生成故事
- **THEN** 3-5 岁默认生成 6 页故事，6-8 岁默认生成 8 页故事，9-12 岁默认生成 10 页故事
- **AND** 每页保持 3-5 句、适合儿童朗读的温暖表达
- **AND** 当年龄缺失时，系统按低龄默认档位生成较短但完整的绘本式故事

---

### Requirement: 故事管理
系统 SHALL 提供故事库管理功能，包括已生成故事列表、推送历史、每日故事记录。

#### Scenario: 查看故事库
- **WHEN** 家长进入管理页面
- **THEN** 展示所有已生成的故事，支持按孩子、日期、主题筛选和搜索

#### Scenario: 查看推送历史
- **WHEN** 家长查看推送历史
- **THEN** 展示已推送给孩子的故事列表及推送时间

#### Scenario: 每日故事记录
- **WHEN** 家长查看孩子的故事记录
- **THEN** 展示孩子每日接收的故事列表，形成阅读日历

---

### Requirement: 社区故事市场
系统 SHALL 提供社区故事市场，家长可以浏览、搜索其他家长分享的故事，免费浏览，下载消耗 Token。

#### Scenario: 浏览社区故事
- **WHEN** 家长进入社区页面
- **THEN** 展示按热度/时间排序的故事列表，支持按年龄段、主题筛选

#### Scenario: 搜索故事
- **WHEN** 家长输入关键词搜索
- **THEN** 返回匹配的故事列表

#### Scenario: 免费浏览
- **WHEN** 家长点击社区中的故事
- **THEN** 可免费查看故事摘要和前几页预览

#### Scenario: Token 下载
- **WHEN** 家长点击"下载完整故事"
- **THEN** 消耗 Token 后解锁完整故事内容

#### Scenario: 已购故事解锁全文
- **WHEN** 家长已下载社区中的某篇故事
- **THEN** 系统为该家长记录已购归属，并允许其后续在已购内容中查看完整故事

#### Scenario: 未购买用户不可查看全文
- **WHEN** 家长尚未下载他人公开故事
- **THEN** 系统仅返回摘要和前几页预览，不返回完整故事内容

---

### Requirement: Token 经济系统
系统 SHALL 实现基础的 Token 经济模型，包括 Token 获取、消耗和收益记录。

#### Scenario: 生成故事消耗 Token
- **WHEN** 家长点击"生成故事"
- **THEN** 根据故事长度和插画数量消耗对应 Token

#### Scenario: 下载他人故事消耗 Token
- **WHEN** 家长下载社区中他人分享的故事
- **THEN** 消耗 Token，部分分配给原作者

#### Scenario: 重复下载幂等
- **WHEN** 家长重复下载同一篇已购故事，或因网络重试重复提交下载请求
- **THEN** 系统不得重复扣减 Token，并直接返回该家长已有的完整访问权限

#### Scenario: 查看 Token 余额
- **WHEN** 家长进入个人中心
- **THEN** 展示 Token 余额、获取/消耗记录

---

### Requirement: 简易儿童端页面
系统 SHALL 提供一个简易的儿童端 Web 页面，用于预览故事阅读体验，支持翻页、AI 朗读功能。

#### Scenario: 故事阅读
- **WHEN** 孩子在儿童端打开推送的故事
- **THEN** 展示图文翻页阅读界面，支持上下/左右翻页

#### Scenario: AI 朗读
- **WHEN** 孩子点击朗读按钮
- **THEN** 系统使用 TTS 语音逐页朗读故事内容

#### Scenario: 收藏与回看
- **WHEN** 孩子收藏故事
- **THEN** 故事加入收藏列表，支持随时回看

#### Scenario: 儿童端独立访问
- **WHEN** 孩子在儿童端访问阅读页面
- **THEN** 系统使用独立于家长登录态的儿童访问凭证，仅展示该孩子被推送的故事

---

### Requirement: 童话绘本化界面体验
系统 SHALL 提供统一的轻童话绘本风界面语言，优先覆盖家长端核心流程，并通过轻量背景动效增强品牌感与沉浸感，同时不影响阅读、输入与移动端性能。

#### Scenario: 家长端统一视觉语言
- **WHEN** 家长访问首页、创建故事、故事库、社区等核心页面
- **THEN** 系统使用统一的绘本风色彩、圆角、阴影、按钮、卡片和表单样式
- **AND** 页面在不同模块之间保持一致的品牌识别与操作节奏

#### Scenario: 轻量背景动效
- **WHEN** 用户停留在主要页面
- **THEN** 系统展示柔和、低干扰的背景动效，如云朵、星屑、光晕等缓动元素
- **AND** 动效不遮挡主要内容，不影响可读性、输入体验和点击操作

#### Scenario: 家长端优先改版
- **WHEN** 系统推进界面重设计
- **THEN** 优先完成首页、创建故事页、故事库和社区页的绘本化升级
- **AND** 儿童端视觉后续在同一设计语言下继续统一

#### Scenario: 阅读详情与家长工作台延展升级
- **WHEN** 系统继续推进第二阶段绘本化改版
- **THEN** 故事详情页应强化绘本阅读沉浸感、翻页节奏与朗读操作体验
- **AND** 家长仪表盘应强化“今晚故事准备台”的任务导向与孩子画像信息层级
- **AND** 个人中心应强化账户资料、Token 资产与流水记录的清晰表达

#### Scenario: 儿童端与档案页第三阶段升级
- **WHEN** 系统继续推进第三阶段绘本化改版
- **THEN** 孩子管理页应强化为“成长档案工坊”，突出档案编辑、当前孩子状态与画像标签层级
- **AND** 儿童端阅读页应强化为“今晚故事剧场”，突出沉浸阅读、朗读陪伴与翻页节奏
- **AND** 儿童端收藏页应强化为“回忆收藏册”，突出收藏、回看与阅读记录的纪念感

#### Scenario: 认证入口第四阶段升级
- **WHEN** 系统继续推进第四阶段绘本化改版
- **THEN** 登录页应强化为“回到今晚的故事准备台”，突出回归、继续创作与快速登录
- **AND** 注册页应强化为“为孩子开启第一本专属童话”，突出启程、品牌价值与温柔引导
- **AND** 认证场景下的全局壳层应进入更安静的轻壳层模式，弱化主导航与移动端底部导航干扰

---

### Requirement: 用户认证系统
系统 SHALL 提供用户注册、登录、密码重置等基础认证功能。

#### Scenario: 注册新账户
- **WHEN** 新用户访问平台
- **THEN** 可通过邮箱或手机号注册账户

#### Scenario: 登录
- **WHEN** 已注册用户访问平台
- **THEN** 可通过邮箱/手机号 + 密码登录

---

### Requirement: 部署与数据库环境切换
系统 SHALL 支持本地开发与 Render 生产部署的双环境运行，并保证 Prisma schema、迁移目录、运行时数据库连接保持一致。

#### Scenario: 本地开发环境使用 SQLite
- **WHEN** 开发者在本地使用 `file:./dev.db` 作为 `DATABASE_URL`
- **THEN** Prisma CLI 自动选择 SQLite schema 与 SQLite 迁移目录，并可正常生成 Client、构建和启动服务

#### Scenario: Render 生产环境使用 PostgreSQL
- **WHEN** 服务部署到 Render，且 `DATABASE_URL` 为 PostgreSQL 连接串
- **THEN** Prisma CLI 自动选择 PostgreSQL schema 与 PostgreSQL 迁移目录，并在启动前成功执行迁移

#### Scenario: 前端部署地址可配置
- **WHEN** 前端构建到不同环境
- **THEN** 系统通过环境变量注入 API 基地址，避免硬编码本地接口地址

---

## 非功能需求
- 页面加载时间 < 3 秒
- 移动端优先的响应式设计
- AI 生成故事常规响应时间 < 60 秒（不含首轮冷启动）
- 支持中英文双语
- 家长端核心页面采用统一的轻童话绘本风设计语言
- 轻量背景动效不显著影响移动端交互流畅度与文本可读性
