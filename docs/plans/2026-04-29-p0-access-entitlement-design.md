# P0 访问与归属修复设计

## 背景

当前项目存在 3 个直接影响业务闭环的 P0 问题：

1. 社区公开故事详情会直接返回全文，未下载用户可以绕过 Token 付费读取完整内容。
2. 儿童端 `/kids` 路由虽然是公开的，但实际依赖家长登录态读取数据，匿名访问会失败。
3. 下载流程只有扣费与收益分账，没有记录“已购归属”，也没有幂等保护，存在重复扣费风险。

本次修复采用方案 B：通过最小但真实的权限与归属模型，统一解决社区预览、下载解锁与儿童端独立访问问题。

## 目标

- 未购用户查看社区故事时只能看到摘要与前几页预览。
- 已购用户可以稳定访问完整故事，并在故事库中看到“已购内容”。
- 同一用户重复下载同一故事不会重复扣减 Token。
- 儿童端使用独立访问凭证读取被推送故事，不再依赖家长登录态。

## 非目标

- 本轮不处理密码重置、手机号登录、阅读日历、流式生成等 P1/P2 问题。
- 本轮不引入复杂订单状态机或事件总线。
- 本轮不将儿童端收藏与阅读记录全面迁移到服务端。

## 核心设计

### 1. 社区预览与完整内容分离

后端对社区列表与社区详情返回预览视图，而不是直接返回 `Story` 原始全文：

- 社区列表：返回标题、摘要、作者、主题、年龄段、封面图、下载次数。
- 社区详情：在上述字段基础上增加前几页预览内容。
- 完整故事内容：仅对作者本人、已购用户开放。

这样可以保留现有 `Story` 作为源数据，同时在接口层做访问投影，避免前端误拿全文。

### 2. 下载归属与幂等

新增两类数据：

- `StoryEntitlement`：记录某用户是否拥有某故事的完整访问权限。
- `DownloadOrder`：记录某用户对某故事的下载行为与幂等键。

规则如下：

- 作者天然拥有自己的故事访问权限，不需要额外 entitlement。
- 用户首次下载公开故事时，在一个事务中完成：
  - 校验余额；
  - 创建或确认下载订单；
  - 扣减买家 Token；
  - 增加作者收益；
  - 创建 `StoryEntitlement`；
  - 增加故事下载次数。
- 若同一用户重复下载同一故事，则直接返回已存在 entitlement，不再扣费。

### 3. 儿童端独立访问模型

新增两类数据：

- `StoryDelivery`：记录故事被推送到哪个孩子。
- `ChildSession`：记录某孩子当前可用的访问凭证。

规则如下：

- 家长推送故事时，不再只设置 `isPushed` 标记，而是写入 `StoryDelivery`。
- 家长可为某个孩子生成或刷新一个儿童访问凭证。
- 儿童端通过访问凭证查询当前孩子被推送的故事列表和详情。
- 儿童端仅能读取与该凭证绑定孩子相关的故事，不暴露家长的私有故事列表。

## 接口调整

### 社区接口

- `GET /api/community/stories`
  - 返回公开故事预览列表。
- `GET /api/community/stories/:id`
  - 返回公开故事预览详情，仅含摘要与前几页。

### 故事接口

- `GET /api/stories/:id`
  - 作者本人或已购用户返回完整内容。
  - 未购用户访问他人公开故事时返回 403，前端应改走社区详情接口。
- `GET /api/stories/purchased`
  - 返回当前用户已购故事列表。
- `POST /api/stories/:id/download`
  - 实现幂等下载与 entitlement 写入。

### 儿童端接口

- `POST /api/kids/sessions`
  - 家长为某个孩子生成访问凭证。
- `GET /api/kids/stories`
  - 基于儿童访问凭证返回该孩子被推送的故事列表。
- `GET /api/kids/stories/:id`
  - 基于儿童访问凭证返回该孩子可阅读的故事详情。

## 前端调整

### 家长端

- 社区页跳转到独立的社区详情页，不再复用“我的故事详情页”。
- 故事库增加“我的创作 / 已购内容”切换。
- 故事详情页只面向作者本人或已购用户。

### 儿童端

- 儿童端启动时从 URL 或本地存储读取儿童访问凭证。
- 读取成功后展示当前孩子被推送的故事书架。
- 若凭证失效或不存在，展示引导信息而不是跳转家长登录页。

## 数据迁移

- 保留现有 `Story.isPushed` 字段以兼容旧数据，但新查询优先使用 `StoryDelivery`。
- 对历史上已经 `isPushed = true` 且绑定了 `childId` 的故事，可在迁移脚本中补写一条 delivery 记录。
- 历史下载数据没有归属记录，本轮不追溯，只对新下载行为生效。

## 风险与取舍

- 社区详情与故事详情拆分后，前端路由和调用链会有一定改动，但权限语义更清晰。
- 老数据无法完整恢复“谁买过什么”，因此已购列表只保证本轮修复上线后的数据准确。
- 儿童访问凭证采用最小化模型，先满足独立访问；后续如需设备管理、撤销登录，再扩展即可。

## 验收标准

- 未下载用户在社区详情页看不到完整 `content`。
- 同一用户重复下载同一故事不会重复扣费。
- 已购用户可在“已购内容”中进入完整故事详情。
- 儿童端在不登录家长账户的情况下可读取被推送故事。
- 自动化测试覆盖社区权限、下载幂等、儿童访问凭证主链路。
