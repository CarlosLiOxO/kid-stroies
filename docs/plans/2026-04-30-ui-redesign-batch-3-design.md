# UI 绘本化重设计第三批设计

## 背景

前两批绘本化重设计已经完成家长端核心主线的统一升级：

- 第一批完成了首页、创建故事页、故事库页、社区页以及全局导航和设计基座。
- 第二批完成了故事详情页、家长仪表盘和个人中心，让家长端从入口、创作、浏览、阅读到账户管理形成了较统一的轻童话绘本风体验。

但 `ChildrenPage`、`KidsPage`、`KidsFavoritesPage` 仍然存在明显的视觉断层：

- `ChildrenPage` 还停留在旧的家长端管理页层级里，更像一页普通表单加列表。
- `KidsPage` 和 `KidsFavoritesPage` 采用一套偏中性的儿童端阅读界面，与当前 fairy 设计系统同源性不足。

用户已经确认，本轮第三批采用“强区分”方案推进，不只是做样式统一，而是让三页在统一品牌母体下分别承担不同角色：

- `ChildrenPage`：成长档案工坊
- `KidsPage`：今晚故事剧场
- `KidsFavoritesPage`：回忆收藏册

## 目标

- 让 `ChildrenPage` 从普通管理页升级为更有“角色设定簿”感的成长档案工坊。
- 让 `KidsPage` 从中性阅读器升级为更有夜读氛围与沉浸感的今晚故事剧场。
- 让 `KidsFavoritesPage` 从普通列表页升级为更有珍藏感的回忆收藏册。
- 继续复用前两批已经落地的 `fairy-*` 语义类，同时补少量第三批所需的儿童端分支语义类。

## 非目标

- 本轮不改动 `ChildrenPage`、`KidsPage`、`KidsFavoritesPage` 的路由结构。
- 本轮不改动孩子档案、收藏、翻页、朗读、本地阅读记录等业务逻辑。
- 本轮不引入儿童端独立状态管理、独立导航系统或新的组件库。
- 本轮不重做登录注册页、儿童访问凭证模型或后端接口。

## 核心设计

### 1. 第三批定位

第三批不是简单延续家长端页面语言，而是在统一品牌体系中做“场景强区分”：

- `ChildrenPage`：成长档案工坊
  - 关键词：建档、设定、整理、长期陪伴。
  - 适合偏纸本档案、角色设定簿的表达。
- `KidsPage`：今晚故事剧场
  - 关键词：月夜、沉浸、翻页、朗读、陪伴。
  - 是第三批里情绪氛围最强的一页。
- `KidsFavoritesPage`：回忆收藏册
  - 关键词：收藏、贴纸、徽章、书签、回看。
  - 比 `KidsPage` 更轻快，更强调纪念感与成就感。

### 2. ChildrenPage 设计方向

`ChildrenPage` 仍属于家长端，因此保持与前两批同源，但要从“管理列表页”升级为“成长档案工坊”：

- 页面外层切换到 `fairy-shell` 与 `fairy-hero`，建立与家长端主线一致的进入感。
- 左栏保持表单编辑逻辑，但视觉上强化为“档案编辑台”：
  - 标题更像档案簿而不是管理表单；
  - 自然语言描述区增加更清晰的引导和示例感；
  - 错误态、编辑态、保存态与取消态统一到 fairy 风格。
- 右栏孩子列表升级为“角色卡册”：
  - 每个孩子卡片更像一张角色设定卡；
  - 强化昵称、年龄、性格、关注点和标签层级；
  - 强化“当前使用中”状态，不只是普通 badge。

### 3. KidsPage 设计方向

`KidsPage` 是第三批最重要的儿童端旗舰页，应从“简易阅读器”升级为“今晚故事剧场”：

- 保留当前左侧选故事、右侧阅读的逻辑结构，避免重写数据流。
- 左侧故事入口从普通按钮网格升级为“书架缩略卡”：
  - 当前选中项更像发光封面；
  - 未选中项弱化干扰，突出封面和标题。
- 右侧阅读舞台继续保留图文双栏，但表达上更像夜读舞台：
  - 标题区更柔和；
  - 朗读按钮成为主陪伴动作；
  - 收藏按钮更像书签而不是普通操作按钮；
  - 翻页控制条更有节奏感。
- 整体色彩比家长端更偏夜读支线，但仍与 fairy 体系同源。

### 4. KidsFavoritesPage 设计方向

`KidsFavoritesPage` 不应再只是“收藏列表/阅读记录”切换，而是更像一个孩子自己的故事小书匣：

- 页面顶部采用更轻量的儿童端头部，不做过重 Hero，但保留清晰的主题表达。
- 收藏 / 阅读记录切换从普通 tab 升级为书签式切换。
- 收藏列表更突出“珍藏感”：
  - 封面、贴纸、徽章、星标和轻量角标。
- 阅读记录更突出“轨迹感”：
  - 露出故事标题、日期、最近状态；
  - 更像在翻一本回忆册，而不是看后台历史列表。

### 5. 样式策略

第三批继续复用现有 `fairy-*` 样式体系，但为儿童端补少量分支语义类：

- `fairy-archive-card`
- `fairy-kids-stage`
- `fairy-bookmark-tab`
- `fairy-memory-card`
- `fairy-sticker-badge`

这些类的目的不是创造一套新的系统，而是在现有设计语言之上，为儿童端和档案页补足更强的场景角色感。

## 前端调整

### 样式层

- 在 `client/src/index.css` 中补齐第三批所需的儿童端和档案页语义类。
- 优先复用现有 `fairy-shell`、`fairy-hero`、`fairy-panel`、`fairy-book-card`、`fairy-chip-*`、`fairy-empty`。
- 仅对儿童端增加少量支线配色与装饰类，避免重新分叉出完整主题系统。

### 页面层

- `ChildrenPage.tsx`
  - 重点改为档案编辑台 + 角色卡册结构。
- `KidsPage.tsx`
  - 重点改为故事剧场式阅读舞台。
- `KidsFavoritesPage.tsx`
  - 重点改为收藏册与回看记录页。

## 风险与取舍

- 如果儿童端过度继承家长端布局，会失去角色感，因此 Kids 页面需要保留更明显的儿童支线表达。
- 如果儿童端完全独立成另一套主题，又会导致品牌割裂，因此必须坚持“同源但强区分”。
- 如果在 `KidsPage` 中加入过重动画，会影响阅读和设备流畅度，因此本轮仍只做轻量装饰和层次增强。
- 如果 `ChildrenPage` 改得太像普通后台编辑页，会削弱第三批的完成度，因此必须强化“成长档案工坊”的角色。

## 验收标准

- `ChildrenPage` 具备更强的成长档案工坊感，编辑区与角色卡区层级清晰。
- `KidsPage` 具备更强的今晚故事剧场感，朗读、翻页、收藏和阅读舞台更有沉浸感。
- `KidsFavoritesPage` 具备更强的回忆收藏册感，收藏与记录切换、卡片表达和空态更有纪念感。
- 三页与前两批保持统一品牌母体，但用户能明显感知三页角色不同。
- 不引入业务链路回退，不改变孩子档案 CRUD、收藏、本地阅读记录、翻页与朗读逻辑。
