# 全站 UI 绘本化重设计设计

## 背景

当前项目已经具备首页、创建故事、故事库、社区、儿童端等主要业务流程，但整体界面仍以通用暖色后台样式为主，缺少鲜明的产品品牌感、绘本氛围与一致的组件语言。用户已经明确确认，本轮需要将产品升级为“轻童话绘本风”，并加入轻量背景动效，让家长端看起来更像一个温暖、柔和、有故事感的童话产品，而不是普通管理后台。

目前的主要问题集中在以下几点：

1. 全局缺少统一的视觉 token，颜色、圆角、阴影、按钮、卡片和输入框虽然已有基础样式，但品牌辨识度不足。
2. 首页、创建故事页、故事库、社区页的页面气质不统一，信息结构更偏功能堆叠，缺少情绪化设计与层级节奏。
3. 背景仍然较平，无法体现轻童话绘本风应有的柔和氛围和空间感。
4. 家长端与儿童端视觉尚未形成同源但有区分的设计系统。

## 目标

- 建立一套统一的轻童话绘本风设计语言，先覆盖家长端核心页面。
- 让首页、创建故事页、故事库、社区页形成一致的品牌感、版式节奏与操作反馈。
- 在不引入新 UI 框架的前提下，基于现有 Tailwind 和全局语义类完成风格升级。
- 加入轻量背景动效，增强沉浸感，但不影响阅读、输入、点击与移动端性能。
- 为后续儿童端视觉统一打好主题基础。

## 非目标

- 本轮不重写业务路由、数据模型或前后端接口。
- 本轮不引入新的设计系统库或组件库。
- 本轮不优先重做儿童端页面，只保留后续统一入口。
- 本轮不追求复杂动画编排、插画生成系统或高成本视觉特效。

## 核心设计

### 1. 设计语言定位

本轮采用“轻童话绘本风”作为统一方向，视觉特征如下：

- 底色以奶油白、浅杏、雾粉、淡紫、浅天蓝为主，避免高饱和儿童玩具风。
- 卡片和容器使用更柔和的大圆角、云朵感边缘与浅层次阴影。
- 标题与重点信息强调温暖、梦幻、亲近感，避免后台式强功能色块。
- 页面保留足够留白，让内容看起来更像绘本排版而不是控制台表格。

该方向优先服务家长端，因为当前产品最核心的操作链路仍是家长创建故事、管理故事和浏览社区。

### 2. 全局主题与背景层

重设计将首先落在全局样式层，而不是每个页面各自零散修改。基础样式将集中在 `client/src/index.css`：

- 统一定义页面背景渐变、浮层光晕、云朵或星屑类装饰。
- 重定义按钮、卡片、输入框、标签、标题、分区容器的语义类。
- 给 hover、focus、active 提供更柔和但明确的反馈。
- 保持现有 Tailwind 原子类可用，但优先让核心页面复用新的语义类。

背景动效采用“低干扰、低频率、低层级”的原则：

- 以缓慢漂浮的云朵、微弱闪烁的星屑、柔和呼吸光晕为主。
- 动效只作为背景层存在，不遮挡正文和表单。
- 在移动端控制数量和透明度，避免滚动卡顿与视觉疲劳。

### 3. 全局布局与导航

全局布局壳由 `AppLayout`、`Header`、`MobileNavigation`、`Footer` 共同承载：

- `AppLayout` 负责统一背景层、页面最大宽度、区块间距与主内容容器。
- `Header` 从“通用顶部栏”升级为具备品牌识别的导航头部，强化产品名、主入口与当前页面状态。
- `MobileNavigation` 延续底部导航模型，但在风格上与整体绘本语言统一，弱化工具感。
- `Footer` 保持克制，不抢夺主页面氛围，但与新主题保持一致。

这样可以优先统一“用户进入任意页面时的第一感知”，并为后续每个页面的内容模块提供稳定外壳。

### 4. 页面改版优先级

本轮页面重设计按以下顺序推进：

1. 首页：建立第一印象、品牌气质和核心行动入口。
2. 创建故事页：作为最核心转化链路，升级为更有仪式感的“故事编织台”。
3. 故事库页：升级为更像绘本书架的浏览体验，强化卡片、筛选与状态信息。
4. 社区页：升级为更像故事集市和灵感广场的内容浏览体验。

后续第二阶段再继续覆盖：

- `StoryDetailPage`，让阅读详情更有绘本翻阅感。
- `DashboardPage`、`ChildrenPage`、`ProfilePage`、登录注册页，统一账户与管理体验。
- `KidsPage` 与 `KidsFavoritesPage`，做儿童端独立但同源的视觉分支。

### 5. 首页设计方向

首页将从“功能入口卡片页”升级为“品牌首页”：

- 首屏增加更完整的品牌标题、副标题、主行动按钮与柔和插图/背景层。
- 快捷入口保留，但会改造成更像故事模块入口而非普通卡片链接。
- 增加产品价值表达，例如“为每个孩子定制专属睡前故事”的情绪型文案。
- 首页整体应兼顾桌面与移动端，移动端优先保留首屏转化效率。

### 6. 创建故事页设计方向

创建故事页从表单页升级为“故事编织台”：

- 输入区、风格选择区、孩子信息区与操作区分成更清晰的视觉模块。
- 强化主行动按钮与生成状态反馈，让用户在等待时仍感受到情绪陪伴。
- 表单输入框、标签和选项卡采用更柔和的绘本样式，减少业务表单的生硬感。
- 保留现有功能结构，降低实现成本。

### 7. 故事库与社区页设计方向

故事库页与社区页将从“信息卡片列表”升级为“内容浏览场”：

- 故事库强调“绘本书架感”，卡片要更像一本本可翻阅的小故事。
- 社区页强调“故事集市感”，在浏览、筛选和探索上更有逛内容的氛围。
- 两个页面共享卡片、筛选条、状态标签、列表节奏，但通过文案和局部样式体现不同场景。
- 保留当前“内容区整块可点、按钮区独立”的交互原则，不回退已有可用性优化。

## 前端调整

### 样式层

- 在 `client/src/index.css` 中补齐主题 token 与全局语义类。
- 为背景层增加可复用的装饰类和轻量动画类。
- 统一按钮、卡片、输入框、标题、分组标题、标签和空态容器样式。

### 布局层

- 调整 `AppLayout.tsx`，统一主容器宽度、背景层与页面留白节奏。
- 调整 `Header.tsx`、`MobileNavigation.tsx`、`Footer.tsx`，让导航系统与新主题匹配。

### 页面层

- 优先改造 `HomePage.tsx`、`CreateStoryPage.tsx`、`StoriesPage.tsx`、`CommunityPage.tsx`。
- 尽量通过抽取复用块或语义类实现升级，避免每页复制大量样式。

## 风险与取舍

- 如果过度追求插画化和动画化，会增加前端实现成本，并可能影响移动端性能，因此本轮只做轻量背景与气质升级。
- 如果只改局部页面、不先改全局主题层，会导致页面间风格继续割裂，因此必须先做设计基座。
- 如果把儿童端一并纳入首批实施，范围会明显扩大，影响交付节奏，因此本轮先家长端、后儿童端。
- 当前项目主要采用页面内 Tailwind 类，重设计时需控制样式分散风险，优先沉淀全局语义类。

## 验收标准

- 首页、创建故事、故事库、社区页具有统一的轻童话绘本风视觉语言。
- 顶部导航、底部导航、按钮、卡片、输入框在核心页面中具备统一家族感。
- 页面背景存在轻量动效，但不遮挡内容、不影响阅读与输入。
- 移动端下核心页面仍保持稳定布局、清晰层级和良好点击区域。
- 已有故事库/社区卡片点击行为、生成故事流程等关键交互不发生回退。
