# UI 绘本化重设计第二批设计

## 背景

第一批绘本化重设计已经完成家长端的全局样式基座、导航壳、首页、创建故事页、故事库页和社区页升级，整体品牌气质已经从通用暖色后台转向轻童话绘本风。但剩余的 `StoryDetailPage`、`DashboardPage`、`ProfilePage` 仍停留在旧的页面语言中，导致用户从首页和列表进入详情页或个人页后，视觉体验会出现明显断层。

这三个页面分别承载不同的高频任务：

1. `StoryDetailPage` 承载完整阅读、朗读、下载和推送，是最适合强化沉浸感的页面。
2. `DashboardPage` 是家长高频进入的总览页，决定家长是否能快速进入“今晚故事准备”状态。
3. `ProfilePage` 负责账号资料、Token 资产和流水记录，需要在统一品牌风格下提供更强的清晰度与信任感。

用户已经确认，本轮第二批重设计以“阅读沉浸”作为优先主轴推进。

## 目标

- 让 `StoryDetailPage` 成为更像绘本阅读台的详情页，强化插画、分页、朗读与操作节奏。
- 让 `DashboardPage` 成为家长端的“今晚故事准备台”，增强总览与下一步行动引导。
- 让 `ProfilePage` 在统一绘本品牌下保持更理性、清晰的账户与资产表达。
- 继续复用第一批已落地的 `fairy-*` 语义类，避免第二批页面形成新的样式分叉。

## 非目标

- 本轮不改动故事详情、仪表盘和个人中心的路由结构。
- 本轮不改动接口、数据库、store 状态结构或业务权限逻辑。
- 本轮不引入复杂翻书动画、图表系统或新的组件库。
- 本轮不优先重做 `ChildrenPage`、登录注册页和儿童端页面。

## 核心设计

### 1. 第二批页面定位

三页在同一品牌语言下做差异化分工：

- `StoryDetailPage`：绘本阅读台
  - 关键词是沉浸、陪伴、翻页与朗读。
  - 视觉装饰密度最高，但仍保持阅读优先。
- `DashboardPage`：今晚故事准备台
  - 关键词是总览、下一步、孩子画像和晚间陪伴节奏。
  - 风格比详情页更克制，但比个人中心更温暖。
- `ProfilePage`：账户与资产小屋
  - 关键词是清晰、可信、秩序。
  - 风格最稳定，减少过强装饰与情绪化元素。

### 2. StoryDetailPage 设计方向

当前详情页已经具备标题、标签、摘要、插画、分页、下载、推送和朗读功能，本轮不改逻辑，只重构阅读感：

- 页头升级为 `fairy-hero`，把标题、作者、适龄信息、摘要和主要操作合并成一块“故事封面信息区”。
- 标签统一改为 `fairy-chip-warm / fairy-chip-lilac / fairy-chip-rose / fairy-chip-sky`，保持与第一批一致。
- 阅读区升级为更像“展开书页”的布局：
  - 左侧是插画页；
  - 右侧是正文页；
  - 中间通过留白、浅阴影和暖纸色增强书页感。
- 页码、朗读、上一页/下一页统一成一套阅读控制条，而不是分散在不同位置。
- 下载结果和错误提示改成统一的 `fairy-message-success / fairy-message-error`。
- 加载态和空态改成更柔和的绘本空白页表现。

### 3. DashboardPage 设计方向

仪表盘不再只是指标卡 + 列表，而是家长每天进入后的“准备台”：

- 顶部改成 `fairy-hero`，通过欢迎语和主 CTA 直接引导家长进入“今晚要为孩子准备什么故事”。
- 统计卡从后台指标卡改成叙事型总览卡：
  - 我的故事
  - 我的孩子
  - 剩余 Token
- 最近生成的故事继续保留，但视觉上更像轻量书架卡而不是普通文章列表。
- 当前孩子画像区应强化角色感，突出姓名、年龄、性格与关注点层级。
- 每日故事记录继续保留，但通过更柔和的卡片和标题层次，让它更像陪伴轨迹，而不是操作日志。

### 4. ProfilePage 设计方向

个人中心需要在统一主题下保持更清晰的秩序感：

- 页面头部改成 `fairy-hero`，展示账户身份与页面作用。
- 左侧账号信息区升级为更完整的账户卡，强调头像、昵称、邮箱、加入时间和当前余额。
- Token 充值区从普通表单块升级为“钱包操作区”，但保留现有模拟充值逻辑。
- Token 流水区改为更像账本的排版：
  - 左侧描述与时间；
  - 右侧金额；
  - 正负金额继续保持颜色区分。
- 页面整体减少高密度装饰，建立更安静、更可靠的账户心智。

## 前端调整

### 复用层

- 优先复用第一批已经落地的 `fairy-shell`、`fairy-stack`、`fairy-hero`、`fairy-panel`、`fairy-chip-*`、`fairy-message-*`、`fairy-empty`。
- 如确有必要，只补少量新的语义类：
  - `fairy-reader-stage`
  - `fairy-metric-card`
  - `fairy-ledger-row`

### 页面层

- `StoryDetailPage.tsx`
  - 重点做头部封面信息区、阅读区和分页控制条升级。
- `DashboardPage.tsx`
  - 重点做 Hero、总览卡、最近故事和孩子画像区升级。
- `ProfilePage.tsx`
  - 重点做账户卡、钱包区和账本式流水区升级。

## 风险与取舍

- 若在详情页加入过重的翻书动画，可能影响移动端性能和阅读连续性，因此本轮只做静态书页感和轻交互反馈。
- 若仪表盘过于追求数据丰富度，会重新回到后台感，因此本轮坚持“任务导向优先、图表从简”。
- 若个人中心过度沿用详情页和首页的梦幻表达，会削弱资产与账户的清晰度，因此 Profile 的装饰密度需明显低于另外两页。
- 当前项目尚未为这三页建立独立测试覆盖，本轮实现时需谨慎决定是否新增聚焦测试，避免低价值测试噪音。

## 验收标准

- `StoryDetailPage` 具备更强的绘本阅读沉浸感，标签、摘要、阅读区和操作区样式与第一批页面统一。
- `DashboardPage` 具备明确的总览与下一步行动引导，页面不再呈现后台式指标堆叠。
- `ProfilePage` 具备清晰的账户资料、Token 余额和账本式流水展示。
- 三页在移动端和桌面端下都与第一批已完成页面保持统一设计语言。
- 不引入业务链路回退，不改变下载、推送、朗读、充值、拉取数据等现有行为。
