# UI 绘本化重设计第四批设计

## 背景

前三批绘本化重设计已经完成以下范围：

- 第一批：首页、创建故事页、故事库、社区页以及全局设计基座与布局壳。
- 第二批：故事详情页、家长仪表盘、个人中心。
- 第三批：孩子管理页、儿童端阅读页、儿童端收藏页。

当前登录后的主流程与儿童端主流程已经基本进入统一的轻童话绘本风，但认证入口仍存在明显断层：

- `LoginPage` 仍是旧版通用卡片式表单页；
- `RegisterPage` 仍是旧版通用卡片式表单页；
- 认证场景沿用完整的主导航与移动端底部导航，入口注意力不够集中。

因此第四批聚焦“认证入口批”，目标不是扩展新业务，而是补齐用户进入产品的第一步，让“首页 -> 登录/注册 -> 登录后主流程”的视觉体验真正闭环。

## 目标

- 让 `LoginPage` 从普通登录页升级为“回到今晚的故事准备台”。
- 让 `RegisterPage` 从普通注册页升级为“为孩子开启第一本专属童话”。
- 让认证场景进入更安静的轻壳层模式，减少导航干扰，同时保留品牌识别和背景氛围。
- 保持认证接口、认证 store 与登录/注册跳转逻辑不变，仅重构视觉层、信息层级和壳层表现。

## 非目标

- 本轮不改动登录、注册接口与表单字段。
- 本轮不实现找回密码、邮箱验证、手机号登录等新认证功能。
- 本轮不重做认证数据流、权限逻辑或路由守卫逻辑。
- 本轮不引入新的组件库或单独的认证主题系统。

## 核心设计

### 1. 第四批定位

第四批采用“认证入口批”策略，不做新业务页面扩张，而是聚焦用户进入产品的第一步：

- `RegisterPage`：为孩子开启第一本专属童话
  - 关键词：启程、第一本、期待感、品牌承诺。
  - 气质应比登录页更有情绪、更有开始感。
- `LoginPage`：回到今晚的故事准备台
  - 关键词：回归、继续、熟悉、安全、快速。
  - 气质应比注册页更克制，强调立即回到主流程。
- `Auth 轻壳层`：安静的前台门廊
  - 关键词：专注、减噪、保留品牌、减少干扰。
  - 不是新布局，而是在认证页场景下让现有壳层进入轻模式。

### 2. RegisterPage 设计方向

`RegisterPage` 是新用户的第一步，应承担“品牌承诺落地页”的角色：

- 页面头部要传达“孩子将拥有属于自己的故事世界”，而不仅是“请填写注册信息”。
- 结构上建议拆成两层：
  - 上层：品牌文案、价值说明、轻插画位或故事承诺；
  - 下层：注册表单卡片。
- 视觉上允许比登录页更丰盈：
  - 更明确的副标题；
  - 更轻的启程型文案；
  - 更适合解释“注册后会得到什么”。

### 3. LoginPage 设计方向

`LoginPage` 应承担“快速回归主流程”的角色：

- 文案重点从“登录系统”改为“欢迎回来，继续今晚的故事准备”。
- 保持更高的效率感，减少冗余说明和装饰密度。
- 结构仍以单卡表单为主，但头部和状态区要更有品牌与陪伴感。

### 4. Auth 轻壳层设计方向

登录页和注册页当前仍在完整主壳层下展示，会带来几个问题：

- 顶部完整主导航会分散表单注意力；
- 移动端底部导航在认证场景中并不必要；
- 页脚过于功能化时，会削弱认证页的专注度。

因此第四批建议在 `/login` 和 `/register` 下启用“轻壳层模式”：

- Header：
  - 保留品牌 Logo 与首页返回入口；
  - 隐藏主导航；
  - 右侧只保留少量必要入口。
- MobileNavigation：
  - 在认证页隐藏。
- Footer：
  - 保留但弱化，文案更轻，避免抢表单注意力。

### 5. 样式策略

第四批继续复用现有 `fairy-*` 体系，不单独创造“auth 专属主题”，只补少量适用于认证场景的语义类，例如：

- `fairy-auth-shell`
- `fairy-auth-card`
- `fairy-auth-hero`
- `fairy-auth-note`

这些类用于表达更集中的认证场景，但底层仍与前三批保持同源。

## 前端调整

### 页面层

- `LoginPage.tsx`
  - 重构为“回归页”，强化欢迎回来、快速进入、错误反馈与跳转引导。
- `RegisterPage.tsx`
  - 重构为“启程页”，强化品牌承诺、首次注册情绪与行动 CTA。

### 壳层与导航层

- `AppLayout.tsx`
  - 为认证场景提供轻壳层分支。
- `Header.tsx`
  - 在认证页隐藏主导航，收束右侧入口。
- `MobileNavigation.tsx`
  - 在认证页直接隐藏。
- `Footer.tsx`
  - 在认证页显示更轻版本或降低信息密度。

### 样式层

- `client/src/index.css`
  - 补少量认证入口相关语义类。

## 风险与取舍

- 如果认证页继续使用完整导航壳层，用户会更容易分心，认证完成率和页面完整度都会受影响。
- 如果认证页被做得过度华丽，会影响登录效率，因此登录页应克制，注册页可以更有情绪。
- 如果单独为认证页再做一整套主题，会破坏前三批形成的统一设计母体，因此必须保持同源。
- 如果在这一批顺手加入找回密码等需求，范围会迅速膨胀，不利于稳定落地，因此本轮必须严格控范围。

## 验收标准

- `LoginPage` 具备更强的“回到今晚故事准备台”氛围，并保持登录流程清晰高效。
- `RegisterPage` 具备更强的“开启第一本专属童话”氛围，并清晰表达品牌价值。
- `/login` 与 `/register` 场景下的 Header、Footer、移动端底部导航进入更安静的轻壳层模式。
- 认证接口、认证 store、登录/注册成功跳转逻辑不发生回退。
- 第四批完成后，首页、认证页、登录后主流程、儿童端主流程形成完整一致的绘本化入口体验。
